View Back-End Repo

Technologies used: HTML, CSS, JavaScript, Bootstrap, Axios, DOM/API, GitHub/GitHub Pages, React, react-abcjs, npm, Express

About

This application is intended for musicians, composers, songwriters, and performers of all kinds. It allows users to quickly create and store sketches of your musical ideas, wherever you are and whenever you need, and without the constraints of complex notation programs.

Additional Notes

As a musician and composer, I've had plenty of experience with music notation software through my academic career and beyond, but these programs required, at the very least, a personal computer with this complex software installed to store, edit, and playback my pieces. This app seeks to solve a major part of this issue: mobility. As far as my own experience, I always carried around a pencil and paper in the hopes of capturing ideas, but this always seemed to be forgotten. With modern mobile technology on the rise, it became obvious that a hole existed that I might be able to fill.

Technologies used: HTML, CSS, JavaScript, SASS, Bootstrap, jQuery/AJAX, Handlebars, MongoDB, Mongoose, Express, Git/GitHub, npm

About

This app gives its users a simple, straightforward way to create and post public events for others to view and RSVP to. Each event is broken down into pieces; a name, a date/time, a location, and an option to provide a description of the event. Personal events made by a user can be edited or deleted as they see fit, and each event includes a view of how many users are attending.

About

This app was built for the modern musician; performers, conductors, hobbyists, and everyone in between. Users can log and store their personal practicing sessions, enabling them to see their progress over time. This front end application, as well as its complementary back end framework, represents my second major project completed as a software engineer, and the first to involve a full stack development process. Building on my prior experience creating a front end application with a supplied back end, I followed a process that was quite similar; learn the structures and behaviors of the languages and programs I was working with, proceed slowly and methodically, and work toward a minimum viable product above all else.

About

Let's cut to the chase; this is an app for playing the age-old, time-honored game of Tic-Tac-Toe. Find a friend, start it up, and play as many rounds as you like! This project represents my first real step into software engineering, so while building it was essential for me to spend time planning the structure of the application, define its pieces and their interactions, and make small, incremental steps rather than have the end goal in mind immediately. Throughout the development process, I referenced many sources of information to familiarize myself with language functions and syntax, best practices for programming, and in order to troubleshoot errors and code bugs. These sources include language documentation, error logging, Google Chrome's Developer Tools, external blogs and internet resources, as well as discussing issues with fellow students and our instructors.
